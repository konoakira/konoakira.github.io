Java 面向对象总结

本文总结了 Java 面向对象的一些知识。

两个重要概念

  • 类（Class）：类是一个抽象的概念，类中包含了数据（称为属性，property 或 attribute，或者成员变量，member variable）和对数据的操作（称为方法，method）。
  • 对象（Object）：对象是一个具体的概念，是类的一种具体表达方式。

三个基本特征

  • 继承（Inheritance）

    Java 是单继承，意味着一个类只能从另一个类继承（被继承的类称为父类，基类或者超类），继承使用关键字 extends 。当生成子类对象时，Java 默认首先调用父类的不带参数的构造方法，然后执行构造方法，生成父类的对象。接下来，再去调用子类的构造方法，生成子类的构造对象。

    ☢️注意以下几点：

    • 构造方法不能被继承
    • 方法和属性可以被继承
    • 子类的构造方法隐式地调用父类不带参数的构造方法
    • 当父类没有不带参数的构造方法时，子类需要使用super来显式地调用父类的构造方法

  • 封装（Encapsulation）

    类包含了数据与方法，将数据与方法放在一个类中就构成了封装。

  • 多态（Polymorphism）

    父类型的引用可以指向子类的对象。

四种访问修饰符

安装开放权限从高到低：

  • public
    • 可以被所以类访问
  • protected
    • 类内部
    • 子类
    • 相同包
  • 默认
    • 类内部
    • 相同包
  • private
    • 类内部

使用规则：

  • 如果不允许外部直接通过 new 来创建对象，比如单例模式，构造方法必须是 private
  • 工具类不允许有 public 或 default 构造方法
  • 类非static 成员变量并且与子类共享，必须是 protected
  • 类非static 成员变量并且仅在本类使用，必须是 private
  • 类 static 成员变量如果仅在本类使用，必须是 private
  • 若是 static 成员变量，必须考虑是否为 final
  • 类成员方法只供内部使用时，必须为 private
  • 类成员方法只对继承类公开时，要限制为 protected

关键字

  • void：如果方法不返回值，那么声明方法的时候使用 void关键字，在方法定义中可以由两种情况实现不返回值：

    • 不使用 return 语句
    • 使用return ,但 return 后面没有任何值或者变量，return 后面只有一个分号，表示退出方法，返回到方法的调用端。使用方式：return;

  • super: 对父类对象的引用。如果子类使用super(...)显式地调用父类的某个构造方法，那么在执行的时候就会寻找与super(...)对应的构造方法，而不会再去寻找父类的不带参数的构造方法。与this(...)一样，super也必须要作为构造方法的第一条执行语句，前面不能有任何可执行语句。

  • this：访问本类实例属性和方法，调用时先在本类中寻找，没有则找父类。单独使用时，表示当前对象。

重载（Overload）与重写（Override）

  • 方法签名：由方法名与参数列表（参数类型与顺序）组成，JVM通过方法签名这个唯一键来决定调用哪个重载方法。

  • 构造方法重写：只需看参数即可。如果一个构造方法中调用另一个调用方法，可以用this(...)的方式调用。

  • 方法重写：方法签名不能相同，而方法名要相同，因此方法重写要求参数列表不同，这里有几层含义：

    • 参数个数不同
    • 或者参数类型不同
    • 或者参数顺序不同

  • 子类于父类的方法方法签名一样，这样我们说子类与父类构成了重写关系

  • 当两个方法形成重写关系时，可以在子类方法中通过super.run()形式调用父类的run()方法，其中super.run()不必放在第一行语句，因此此时父类 对象已经构造完毕，先调用父类的run()方法还是先调用子类的run()方法是根据程序的逻辑决定的。

  • 重写时子类的返回类型要比父类更严格，比如父类返回类型是List<String>，则子类可以是ArrayList<String>;子类方法的返回权限要比父类宽松，比如父类是procted则子类只能是public。

  • 方法重载和重写的关系

  • 重载发生在同一个类内部的两个或者多个方法
  • 重写发生在父类和子类之间

抽象类与接口

  • **抽象类(abstract class)**： 使用了abstract 关键字所修饰的类叫做抽象类。抽象类无法实例化，也就是说，不能 new 出来一个抽象类的对象(实例)

    抽象方法: 使用abstract 关键字所修饰的方法叫做抽象方法。抽象方法需要定义在抽象类中。相对于抽象方法，之前所定义的方法叫做具体方法(有声明，有实现)

    如果一个类包含了抽象方法，那么这个类一定是抽象类。如果某个类是抽象类，那么该类可以包含具体方法(有声明，有实现)。无论何种情况，只要一个类是抽象类，那么这个类就无法实例化。在子类继承父类(父类是个抽象类)的 情况下，那么该子类必须要实现父类中所定义的所有抽象方法；否则，该子类也要声明成一个 abstract class

    抽象类体现的就是一种模板模式的设计，抽象类作为多个子类的通用模板，子类在抽象类的基础是上进行扩展、改造

  • 接口(interface): 接口的地位等同于class，接口中所有的方法都是抽象方法。在声明接口中的方法时，可以使用abstract关键字，也可以不使用，通常情况下，都会省略 abstract 关键字

    可以将接口看作成特殊的抽象类，而接口中只能有抽象方法，不能有具体方法(JDK8及以上可以定义默认的具体方法)。类可以实现接口，实现使用关键字 implements 表示，代表了某个实现类实现了某个接口

    一个类实现了某个接口，那么该类必须要实现接口中声明的所有方法。如果该类是一个抽象类，那么就无需实现接口中的方法了

    Java是单继承的，也就是说某个类只能有唯一的一个父类，一个类可以实现多个接口，多个接口之间使用逗号分隔

  • 抽象类与接口最大的不同应该是语义的不同：抽象类是对一个类的模版抽象，而接口是对类的行为的抽象。当然除此之外，一个类可以单继承抽象类，却可以实现多个接口;抽象类可以有非静态成员，接口只能有静态final成员。如果理解了二者的设计目的，也就能理解这些区别

static关键字

  • 修饰属性

    无论一个类生成了多少对象，所有这些对象共同使用唯一一份静态的成员变量；一个对象对静态成员变量进行修改，其他对象的该静态变量的值也会随之发生改变。如果一个成员变量是 static 的，那么我们可以通过 类名.成员变量的方式来使用它

  • 修饰方法

    static 修饰的方法叫做静态方法。对于静态方法来说，可以使用 类名.方法名 的方式来访问; 不能再静态方法中访问非静态成员变量，可以在静态方法中访问静态的成员变量

  • 修饰类

    注意：只能修饰内部类

  • 修饰代码块

    静态代码块的作用也是完成一些初始化工作。首先执行静态代码块，然后执行构造方法。静态代码块在类被加载的时候执行，而构造方法是在生成对象的时候执行；要想调用某个类来生成对象，首先需要将类加载到java 虚拟机上（JVM),然后由JVM加载这个类来生成对象

    类的静态代码块只会执行一次，是在类被加载的时候执行的，因为每个类只会被加载一次，所以静态代码块也会被执行一次；而构造方法则不然，每次生成一个对象的时候都会 调用类的构造方法，所以new 一次就会调用构造方法一次

    如果集成体系中既有构造方法，又有静态代码块，那么首先执行最顶层的类的静态代码块，一直执行到最底层的静态 代码块，然后再去执行最顶层的类的构造方法，一直执行到最底层类的构造方法

final 关键字

  • 修饰属性

    表示不能被改写。当final修饰一个原生数据类型是，表示该原生类型的值不能发生变化。如果final 修饰的是一个引用类型时，表示该引用类型不能再指向其他对象了，但该引用所指向对象的内容是可以发生变化的。成员变量要么声明时赋值，要么在类的构造方法中赋值

  • 修饰方法

    表示不能被重写

  • 修饰类

    表示不能被继承

new 关键字

Cat c = new Cat();语句初始化对象的三个步骤：

  1. 给Cat对象分配内存
  2. 调用Cat类的构造方法
  3. 将c对象指向分配的内存空间

成员变量与局部变量

  • 成员变量
    • 实例变量，不以 static 修饰
    • 类变量，以 static 修饰
  • 实例变量
    • 形参
    • 方法局部变量
    • 代码块局部变量

成员变量使用规则

  • 何时使用成员变量
    • 如果需要定义的变量是用于描述某个类或者某个对象的固有信息
    • 如果在某个类中需要以一个变量来保存该类或者实例运行时的状态信息
    • 如果某个信息需要在某个类的多个方法之间共享
  • 成员变量危害
    • 增大变量的生存时间，将导致更大的内存开销
    • 扩大变量的作用域，不利于提高程序的内聚性

类关系

  • 继承，extends，is-a
  • 实现，implements，can-do
  • 组合，类是成员变量，contains-a，具有相同的生命周期，是一种非常强的整体与部分的关系
  • 聚合，类是成员变量，has-a，具有不同的生命周期，是一种松散的整体与部分的关系
  • 依赖，import 类，use-a

设计父类规则

  • 尽量隐藏父类的内部数据，尽量把父类的所有成员变量都设置成 private 访问类型，不要让子类直接访问父类的成员变量
  • 不要让子类可以随意访问、修改父类的方法。父类中那些仅为辅助其他的工具方法，应该使用 private 访问控制符修饰，让子类无法访问改方法。如果父类中方法需要被外部类调用，则必须以 public 修饰，但又不希望子类重写该方法，可以使用 final 修饰。如果希望父类的某个方法可以被子类重写，但不希望被其他类方法，可以用 protected 修饰该方法
  • 尽量不要在父类构造器中调用将被子类重写的方法

对象的引用

  • 强引用（Strong Reference）

    程序创建一个对象，并把这个对象赋给一个引用变量，程序通过该引用变量来操作实际的对象。当一个对象被一个或一个以上的引用变量所引用时，它处于可达状态，不可能被系统垃圾回收机制回收

  • 软引用（Soft Reference）

    当一个对象只有软引用时，它有可能被垃圾回收机制回收。对于只有软引用的对象而言，当系统内存空间足够时，它不会被系统回收，程序也可使用该对象；但系统内存空间不足时，系统可能会回收它。软引用通常用于对内存敏感的程序中

  • 弱引用（Weak Reference）

    对于只有弱引用的对象而言，当系统垃圾回收机制运行时，不管系统内存是否足够，总会回收该对象所占用的内存

  • 虚引用（幻象引用，Phantom Reference）

    幻象引用仅仅是提供了一种确保对象被finalize以后，做某些事情的机制，比如，通常用来做所谓的Post-Mortem清理机制

类内方法

  • 实例方法（非静态方法），依附于实例对象，可以通过引用变量调用。当.class字节码文件加载之后，实例方法并不会被分配方法入口地址，只有在对象创建之后才会被分配
  • 类方法（静态方法），不能使用实例成员。当类加载后，即分配了相应的内存空间。静态方法不能使用super和this关键字，这两个关键字指代的都是需要被创建出来的对象。通常静态方法用于定义工具类中的方法
  • 静态代码块，不存在于任何方法体内。在类加载的时候就被调用，并先于构造方法执行，且只能执行一次

对象创建的几种方法

  • new，调用构造方法
  • clone，在内存上
  • 反射，调用构造方法
  • 序列化，在文件上
  • 通过Unsafe实例化一个类

getter&setter

  • 满足面向对象语言封装的特性，尽可能将类中的属性定义为 private。针对属性值的访问与修改需要使用相应的getter 与 setter 方法，而不是直接public的属性进行读取和修改。

  • 有利于统一控制，业务要求对某个属性值的修改要增加统一的权限控制，如果有 setter 作为统一的属性修改方法则更容易实现。
