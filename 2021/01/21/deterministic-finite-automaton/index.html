请实现一个函数用来判断字符串是否表示数值(包括整数和小数).例如，字符串”+100”,”5e2”,”-123”,”3.1416”,”-1E-16”,”0123”都表示数值，但”12e”,”1a3.14”,”1.2.3”,”+-5”及”12e+5.4”都不是.

这题很好👍, 它考验的不仅是细心, 还有耐心. 第一遍我暴力条件判断. 只要思维严谨都能过的. 然后官方题解带我走入一个新的世界! 计算机的世界是二元的世界, 非对即错, 非有即无. 对与错, 1与0, 存在与非存在的同构关系, 让它们可以互相替换!

举个例子:

在一组数0, 1, 2, 3中选出两个数 a和b, 返回较大的数. 显然可以这么写

显然就存在这样一个等价关系:
$$
f(a,b) = a > b \Longleftrightarrow largerMap[a][b] = g(a, b)
$$

左边的对错判断就变成了右边的存在判断(如果定义为map, 判断键是否存在)或查询(如果定义为数组, 查询数组元素)判断了!

再进一步!🚀

如果我把a和b看成某两个状态, 把>看出一个状态转移的符号, a > b表示状态转移是否成功, 而我们只允许这样的转移发生: 只能由较大的数转移到较小的数.
因为这里的状态是有限且确定的, 所以我可以模仿上面同构出一个map或者数组. 那么判断状态能否发生转移就很明确了: 当前状态为a, 输入状态b, 如果map[a][b]为true则发生转移.

一个字符在字符串中可能有这样几个状态:

graph LR
A(起始空格) --> |空格| A
A --> |小数点| E
A --> |整数| C
A --> |+/-| B
B(整数符号部分) --> |整数| C
B --> |小数点| E
C(整数部分) --> |整数| C
C --> |e| F
C --> |小数点| D
C --> |空格| J
D(整数后小数点) --> |整数| I
D --> |空格| J
D --> |e| F
E(起始小数点) --> |整数| I
F(e) --> |+/-| G
F --> |整数| H
G(指数符号) --> |整数| H
H(指数后整数) --> |整数| H
H --> |空格| J
I(小数部分) --> |空格| J
I --> |整数| I
I --> |e| F
J(末尾空格) --> |空格| J
